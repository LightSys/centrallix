label

type: widget/label

visual: yes

container: no

form element: yes

Overview:

The label widget is used to display a non-editable string of text. It displays as a simple label.

A label can be used as a form element; to do this, specify a fieldname, and optionally a form.

A label can behave like a "link": to do this, specify a point_fgcolor and a click_fgcolor.

Usage:

The label widget can be placed inside of any visual container, and will attach itself to any form widget that contains it (whether directly or indirectly). Labels may not contain visual widgets.

Properties:

Property Type Description
align string Describes how the text should align in the label (e.g. right).
allow_break string It determines if the text can wrap around something.
click_fgcolor string The color (named or #numeric) of the text in the label when the user clicks on the label.
fgcolor string The color (named or #numeric) of the text in the label.
fieldname string The name of the column in the datasource that will be used to supply the text of the label.
font_size integer (e.g. 16, default: 12).
form string The name of the form that this label is associated with.
height string The height of the label.
overflow_ellipsis string If text is no longer able to display shows an ellipsis.
point_fgcolor string The color (named or #numeric) of the text in the label when the user hovers the mouse over the label.
style string (e.g. bold).
text string The text that the label is to display.
tooltip string The text displayed when the courser hovers over the widget.
valign string Describes how the text should align vertically within the label (e.g. top, middle, or bottom).
width integer The width of the label.
x integer X-coordinate of the upper left corner of the checkbox, default is 0.
y integer Y-coordinate of the upper left corner of the checkbox, default is 0.

Actions:

Action Description
SetValue Sets the value property to the given parameter.

Events:

Event Description
Click This event occurs when the user clicks the checkbox. No parameters are available from this event.
DataChange This event occurs when the user has modified the data value of the checkbox (clicked or unclicked it).
MouseDown This event occurs when the user presses the mouse button on the checkbox. This differs from the 'Click' event in that the user must actually press and release the mouse button on the checkbox for a Click event to fire, whereas simply pressing the mouse button down will cause the MouseDown event to fire.
MouseMove This event occurs when the user moves the mouse pointer while it is over the checkbox. The event will repeatedly fire each time the pointer moves.
MouseOut This event occurs when the user moves the mouse pointer off of the checkbox.
MouseOver This event occurs when the user first moves the mouse pointer over the checkbox. It will not occur again until the user moves the mouse off of the checkbox and then back over it again.
MouseUp This event occurs when the user releases the mouse button on the checkbox.

Client Properties:

Property Type Description
value string This property allows a runclient() expression to dynamically supply the text to display in the label.

none currently available

Sample Code:


		

$Version=2$

// Here is a label.
