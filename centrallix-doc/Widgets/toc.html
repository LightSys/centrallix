Centrallix Application
Widget
Reference


This guide contains a basic overview and specification of each of the widgets that Centrallix supports, including a handful of planned (but not yet implemented or not fully implemented) widgets.

Each widget can have properties, child properties, events, and actions. Properties define how the widget behaves, appears, and is constructed. Child properties usually define how a widget contained within the given widget relates to its parent. Such properties are normally managed by the containing (parent) widget rather than by the child, and many times the child widget cannot exist without its parent. Events enable a widget to have an "influence" on the other widgets on the page, and are utilized by placing a "connector" widget within the given widget. The connector routes an event's firing to the activation of an "action" on some other widget on the page. Actions are "methods" on widgets which cause the widget to do something or change its appearance in some way.

The "overview" section for each widget describes the widget's purpose and how it operates. A "usage" section is provided to show the appropriate contexts for the use of the widget in a real application.

Some widget properties are 'dynamic'; that is, an expression can be provided which is then evaluated dynamically on the client so that the widget's property changes as the expression's value changes.

"Client properties" are available for some widgets. These properties may not be specified in the application file itself, but are available on the client during application operation. They often relate to the status of a given widget.

Where possible, sample code is provided for each widget to show how the widget can be used. In most cases the widget's code is displayed in isolation, although in practice widgets can never be used outside of a "widget/page" container widget at some level. In a few cases, a more complete mini-application is shown.

Sample code is generally given in "structure file" format, which is the normal format for the building of applications. However, other suitable object-structured data formats can be used, including XML.

Documentation on the following widgets is available:

autolayout Container which automatically positions its children
button A multipurpose button
calendar Calendar-type view of event/schedule type data in an objectsource
chart Chart widget for graphing data from an ObjectSource, using the ChartJS library.
checkbox Form element capable of selecting an on/off, yes/no, true/false, etc., type of value, via a visual 'check mark'.
childwindow A dialog or application window which is a lightweight movable container.
clock A simple clock widget which displays the current time on the client.
component A widget which instantiates a custom component
component-decl A widget which defines a custom component
connector A nonvisual widget used to trigger an Action on a widget when an Event on another widget fires.
datetime A visual widget for displaying and editing a date/time type value.
dropdown A visual widget which allows the user to select one from a number of options in a list which appears when the user clicks on the widget.
editbox A visual widget which allows the entry of one line of textual data.
execmethod A nonvisual widget which can call ObjectSystem methods on server objects.
form A nonvisual container used to group a set of form element widgets into a single record or object
formstatus A specialized visual widget used to display the current mode of a form widget.
frameset A visual container used to create a DHTML frameset within which page widgets can be placed
hbox Container which automatically positions its children
hints Contains default values and modifier settings for various components
html A miniature HTML browser control, capable of viewing and navigating simple web documents.
image A picture (image).
imagebutton A button widget which uses a set of images to control its appearance.
label Form element capable of displaying text in a child window
menu A visual pop-up or drop-down menu widget.
osrc A nonvisual widget which handles data communication between forms/tables and the Centrallix server.
page The top-level container that represents a Centrallix application.
pane A visible rectangular container widget with a border
parameter A declaration of a parameter that can be passed to an application or component
radiobuttonpanel A visual widget displaying a set of 'radio buttons'
remotectl Nonvisual widget permitting events to be passed in from a remote Centrallix application
remotemgr Nonvisual widget permitting the application to send events to a remote Centrallix application
repeat Repeat a subtree of widgets
rule Declares a rule - a behavior expected from a widget.
scrollbar A scrollbar with a thumb
scrollpane A visual container with a scrollbar allowing for content height that exceeds the display area
tab A tab (or notebook) widget allowing multiple pages to be layered and individually selected for viewing
table A visual widget providing a columnar presentation of multiple objects of the same type (such as query result records).
template templates are a way to provide default values (and default children) of different widget_class's
textarea Visual multi-line text data entry widget.
textbutton A simple visual button widget built not from images but from a simple text string.
timer A nonvisual widget which is used to fire an event when a period of time has elapsed.
treeview A visual widget used to display tree-structured data (such as a directory tree or similar).
variable A scalar variable object
vbox Container which automatically positions its children
