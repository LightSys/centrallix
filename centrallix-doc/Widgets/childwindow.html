childwindow

type: widget/childwindow

visual: yes

container: yes

form element: no

Overview:

The childwindow provides the capability of creating a popup dialog or application window within a web page. The window is not actually a separate browser window, but rather a movable container that can appear "floating" above the rest of the Centrallix application.They can take on one of two styles;"dialog" and "window", which currently relate only to how the window border and titlebar are drawn. The windows support windowshading (double-clicking on the titlebar to "rollup" or "rolldown" the window contents).

These "windows" currently do not support multiple instantiation.

Usage:

Childwindows are normally coded at the top-level of an application, since if they are placed within containers, the container limits where the window can be moved (the window is clipped by its container). However, if the "toplevel" property is enabled, then the window floats at the top level of other widgets regardless of the size and location of its container in the application or component (thus the window is not clipped by its container).

These windows can contain other visual and nonvisual widgets.

Properties:

Property Type Description
background string A background image for the body of the window.
bgcolor string A color, RGB or named, to be used as the window body's background.If neither transparent.
border_color string A color that outlines window.
border_radius integer A radius that describes the sharpness of the corners of the window (smaller means sharper).
border_style string Determines the look of the outline.
closetype string Decides wither the screen closes widthwise (1) or heightwise (2).
gshade string Acts as a boolean to declare if the window is shaded or not (all but title bar minimized).
hdr_background string A background image for the titlebar of the window.
hdr_bgcolor string A color, RGB or named, for the titlebar of the window.
height integer Height, in pixels, of the window.
icon string A pathname to an icon to be used in the upper left corner of the window. If unspecified, the default "CX" ichthus icon is used instead.
modal yes/no If "yes", the window is modal (and IsModal need not be passed to "Open"). A modal window will force the user to close the window before anything else in the application can be accessed.
shadow_angle integer The placement of the shadow described as a rotational transformation with respect to the window.
shadow_color string The color of the shadow.
shadow_radius integer A radius that describes the sharpness of the corners of the shadow (smaller means sharper).
shadow_offset integer The placement of the shadow with respect to the window.
style string Either "dialog" or "window", and determines the style of the window's border.
textcolor string The color for the titlebar's text (window title). Default "black".
title string The window's title.
titlebar yes/no Whether the window will have a titlebar (and the close "X" in the upper right corner of the window). Default "yes".
toplevel yes/no If "yes", the window will float above all other widgets in the application, otherwise it will be clipped by its own container.
visible boolean The window is initially visible on screen. The window has an action which can "true".
width integer Width, in pixels, of the window.
x integer X-coordinate of the upper left corner of the window, relative to its container.
y integer Y-coordinate of the upper left corner of the window, relative to its container.

Actions:

Action Description
Close Closes the window. Note that widgets inside the window may choose to veto the Close operation: for example, if there is unsaved data in a form.
Open Opens the window. If the parameter IsModal is set to 1, then the window becomes modal (only the window's contents are accessible to the user until the window is closed). If the parameter NoClose is set to 1, then the close button in the upper right corner of the window becomes inactive and the window will only close via the Close, SetVisibility, and ToggleVisibility actions.
Point Makes the window relocate to a side using a triangle (pop over).
Popup Opens a window like a pop-up.
SetVisibility One parameter, "is_visible", which is set to 0 or 1 to hide or show the window, respectively.
Shade Turns the gshade property(see above) on.
ToggleVisibility If the window is visible, this closes it. If it is closed, this opens it.
Unshade Turns the gshade property(see above) off.

Events:

Event Description
Close This event is triggered each time the window is closed (becomes invisible).
Load This event is triggered the first time the window is opened. If the window is visible by default, then this event is triggered when the application loads.
MouseDown This event occurs when the user presses the mouse button on the checkbox. This differs from the 'Click' event in that the user must actually press and release the mouse button on the checkbox for a Click event to fire, whereas simply pressing the mouse button down will cause the MouseDown event to fire.
MouseMove This event occurs when the user moves the mouse pointer while it is over the checkbox. The event will repeatedly fire each time the pointer moves.
MouseOut This event occurs when the user moves the mouse pointer off of the checkbox.
MouseOver This event occurs when the user first moves the mouse pointer over the checkbox. It will not occur again until the user moves the mouse off of the checkbox and then back over it again.
MouseUp This event occurs when the user releases the mouse button on the checkbox.
Open This event is triggered each time the window is opened (becomes visible).

Sample Code:


		

$Version=2$

MyWindow "widget/childwindow"

	{

	x = 10; y = 10;

	width = 200; height = 200;

	style = "dialog";

	bgcolor = "#c0c0c0";

	hdr_background = "/sys/images/grey_gradient2.png";

	title = "An HTML Window";

	textcolor = "black";

	}
